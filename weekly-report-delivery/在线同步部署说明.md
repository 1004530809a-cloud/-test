# 在线同步部署说明

## 这版已经实现了什么

- 网页可通过 `/api/report-state` 读取在线版周报数据。
- 网页和接口都可以通过 `/api/feishu-config-check` 自检飞书快照配置，确认 `PDF快照` 字段是否真的是附件类型。
- 低毛利订单 `原因` 在网页选择后，会调用 `/api/reasons` 在线保存。
- 开启飞书配置后，网页保存会直接回写 `低毛利反馈表`，并同步更新 `周报快照`。
- 未开启飞书配置时，会退回本地 `runtime/online-reasons.json` 存储，方便先本机联调。
- 页面会按 `pollIntervalMs` 自动拉取最新在线数据，实现多人协作时的近实时刷新。
- 如果浏览器里留有“未同步草稿”，页面会优先保住线上共享内容，并提示用户手动点“恢复本地草稿”，避免旧缓存直接覆盖线上版本。

## 本地启动

1. 复制在线配置模板：

```bash
cp /Users/sup/Desktop/周报测试/weekly-report-delivery/weekly-report-online-config.template.json /Users/sup/Desktop/周报测试/weekly-report-delivery/weekly-report-online-config.json
```

2. 如果要联动飞书，再确认：

- [feishu_sync_config.json](/Users/sup/Desktop/周报测试/weekly-report-delivery/feishu_sync_config.json)
- 低毛利反馈表建议补充字段：
  - `原因修改人`
  - `原因修改时间`

3. 启动服务：

```bash
cd /Users/sup/Desktop/周报测试/weekly-report-delivery
python3 online_sync_server.py --port 8765
```

4. 浏览器打开：

```text
http://127.0.0.1:8765/weekly-report.html
```

## 配置说明

在线页面读取 [weekly-report-online-config.json](/Users/sup/Desktop/周报测试/weekly-report-delivery/weekly-report-online-config.json)：

- `enabled`
  - 是否开启在线同步模式。
- `apiBaseUrl`
  - 如果网页和接口同域部署，填空字符串即可。
  - 如果接口单独部署，填完整地址，例如 `https://your-domain.example.com`。
- `pollIntervalMs`
  - 自动拉取最新原因的轮询间隔，默认 `20000`。
- `editorName`
  - 网页提交原因时默认记录的填写人名称。

## 飞书联动说明

在线服务启用飞书配置后，保存原因的链路是：

1. 网页选择低毛利订单原因
2. 在线服务写入飞书 `低毛利反馈表`
3. 在线服务重新汇总原因统计
4. 在线服务同步 `周报快照`
5. 网页自动刷新最新汇总

其中 `周报快照` 的 `原因汇总` 已一并更新。

正式上线前，建议先做一次配置检查：

1. 打开网页后点击“检查飞书快照配置”
2. 或直接访问 `/api/feishu-config-check`
3. 确认返回结果里：
   - `configured=true`
   - `storageMode=bitable_attachment` 时，`attachmentField=PDF快照`
   - `attachmentFieldTypeLabel=附件`

另外，`/api/health` 和 `/api/report-state` 现在都会返回 `build` 字段，里面包含：

- `buildId`
- `generatedAt`
- `files`

如果怀疑 Render 还在跑旧代码，先看这个 `buildId` 是否与本地待发布包一致，再决定是否需要再次 Redeploy。

## 部署建议

这套服务是单个 Python HTTP 服务，不依赖额外三方库，适合部署到支持 Python Web Service 的轻量平台。

部署时至少要保证：

- 工作目录指向 [weekly-report-delivery](/Users/sup/Desktop/周报测试/weekly-report-delivery)
- 启动命令为 `python3 online_sync_server.py --port $PORT`
- 同目录放置：
  - [weekly-report-data.generated.json](/Users/sup/Desktop/周报测试/weekly-report-delivery/weekly-report-data.generated.json)
  - [weekly-report-online-config.json](/Users/sup/Desktop/周报测试/weekly-report-delivery/weekly-report-online-config.json)
  - [feishu_sync_config.json](/Users/sup/Desktop/周报测试/weekly-report-delivery/feishu_sync_config.json)（如需飞书联动）

如果网页和接口同域部署，正式入口直接使用部署平台分配的固定服务地址即可，不需要再额外找 webhook 或临时公网转发。

## 当前边界

- 在线同步目前只覆盖低毛利 `原因` 及其汇总，不会把整份周报所有文案编辑都在线化。
- `确认订单 / 执行订单` 重新生成仍建议在线下执行，再替换生成后的 JSON。
- 如果部署环境不能持久化本地文件，请务必启用飞书配置，让飞书成为在线原因主存储。
